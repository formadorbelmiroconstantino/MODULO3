Cabeçalho Fixo em Tabelas Grandes

Cabeçalhos fixos em tabelas grandes é essencial para usabilidade e performance.

Vou explicar passo a passo, só com HTML + CSS puro (sem JS), depois deixaremos boas práticas profissionais e armadilhas comuns.

1. Ideia principal

O truque é simples:

  • A tabela fica dentro de um container com scroll
  • O <th> usa

    • position: sticky;
    top: 0;
  • Assim, quando fazes scroll vertical, o cabeçalho fica preso no topo.

3. CSS essencial (comentado linha a linha)

/* Container que cria o scroll */
.table-container {
max-height: 400px; /* Altura visível da tabela */
overflow-y: auto; /* Scroll vertical */
border: 1px solid #ccc; /* Apenas estética */
}

/* Tabela ocupa toda a largura */
table {
width: 100%;
border-collapse: collapse;
}

/* Células */
th, td {
padding: 10px;
border: 1px solid #ddd;
text-align: left;
}

/* Cabeçalho fixo */
thead th {
position: sticky; /* Torna o cabeçalho "colável" */
top: 0; /* Fixa no topo do container */
background: #f5f5f5; /* Fundo obrigatório */
z-index: 2; /* Fica acima das linhas */
}

/* Zebra striping (opcional, mas profissional) */
tbody tr:nth-child(even) {
background: #fafafa;
}

4. Resultado visual

  • Scroll vertical funciona
  • Cabeçalho nunca desaparece
  • Sem JavaScript
  • Compatível com browsers modernos

5. Erros comuns (Muito importantes)

  • overflow na table:

    • table {
    overflow: auto; ❌
    }

  • Não funciona com position: sticky

6. Cabeçalhos fixos + scroll horizontal

  • Para tabelas largas:
    • .table-container {
    max-height: 400px;
    overflow: auto; /* vertical + horizontal */
    }
  • Funciona perfeitamente com colunas largas.

7. Quando isso NÃO é Suficiente

Se a tabela tiver:

  • 5.000+
  • 10.000+
  • 50.000+ linhas
    • CSS sozinho não resolve performance.

Nesse caso, o caminho profissional é:

  • Paginação
  • Lazy loading
  • Server-side processing

Normalmente com:

  • DataTables
  • AJAX
  • PHP + MySQL
    • (Foi por isso que já vimos o DataTables. Com a implementação do DataTables, excluimos o trabalho de estilizarmos as tabelas com CSS.)
