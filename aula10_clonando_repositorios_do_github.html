Clonando Repositório do GitHub tendo o GitHub Desktop Instaldo no PC

Clonar um repositório no GitHub significa criar uma cópia local completa (com todos os arquivos, pastas e histórico de commits) de um repositório remoto (que está no GitHub.com) para o seu computador, permitindo que você trabalhe nele offline e depois sincronize as mudanças com o original, seja para contribuir ou fazer suas próprias alterações.

Por que clonar?

  • Trabalho local: Você pode fazer modificações, adicionar arquivos e testar sem afetar o projeto original.
  • Histórico completo: Você obtém o histórico de todas as versões de cada arquivo, facilitando o acompanhamento das mudanças.
  • Contribuição: É o primeiro passo para colaborar em projectos de outras pessoas, baixando o código para desenvolver e depois enviando suas alterações.

Passos para Clonar Repositório do GitHub

Diferença entre Clone e Fork:

  • Clone: Cria uma cópia direta de um repositório ao qual você tem acesso (ou pode contribuir diretamente), mantendo uma ligação com o original para enviar (push) suas alterações.
  • Fork (Bifurcação): Cria uma cópia do repositório na sua própria conta do GitHub, ideal para projetos de outras pessoas, permitindo que você altere livremente e depois sugira suas mudanças via Pull Request (PR) para o repositório original.
