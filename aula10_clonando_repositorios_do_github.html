Clonando Repositório do GitHub tendo o GitHub Desktop Instaldo no PC

Clonar um repositório no GitHub significa criar uma cópia local completa (com todos os arquivos, pastas e histórico de commits) de um repositório remoto (que está no GitHub.com) para o seu computador, permitindo que você trabalhe nele offline e depois sincronize as mudanças com o original, seja para contribuir ou fazer suas próprias alterações.

Por que clonar?

  • Trabalho local: Você pode fazer modificações, adicionar arquivos e testar sem afetar o projeto original.
  • Histórico completo: Você obtém o histórico de todas as versões de cada arquivo, facilitando o acompanhamento das mudanças.
  • Contribuição: É o primeiro passo para colaborar em projectos de outras pessoas, baixando o código para desenvolver e depois enviando suas alterações.

Passos para Clonar Repositório do GitHub

Para clonar um repositório com o GitHub Desktop, acesse a página do repositório no GitHub.com, clique no botão "Code", seleccione "Open with GitHub Desktop", escolha ou crie a pasta local onde vai salvar e clique em "Clone"; ou, directamente no aplicativo, vá em Arquivo > Clonar Repositório, seleccione o repositório da sua lista ou cole a URL, defina o caminho local e clone, sendo os dois métodos eficientes para ter o código no seu PC.

Método 1: Pelo Navegador (Recomendado)

  1. Navegue: Vá até a página do repositório que deseja clonar no seu navegador, como o Repositório de exemplo.
  2. Clique em "Code": Acima da lista de arquivos, clique no botão verde "Code".
  3. Abrir com GitHub Desktop: Clique na opção "Open with GitHub Desktop".
  4. Escolha o Local: Uma janela do GitHub Desktop abrirá. Clique em "Choose..." para seleccionar ou criar a pasta onde o repositório será clonado.
  5. Clone: Clique em "Clone" para finalizar o processo, e o repositório será baixado para sua máquina.

Método 2: Pelo Aplicativo GitHub Desktop

  1. Abra o GitHub Desktop: Inicie o aplicativo em seu computador.
  2. Menu Arquivo: No menu superior, clique em "File" (Arquivo) e seleccione "Clone Repository..." (Clonar Repositório...).
  3. Seleccione a Origem: Clique na aba correspondente (GitHub.com, URL, etc.) e escolha o repositório na sua lista ou insira a URL manualmente.
  4. Defina o Caminho Local: Ao lado de "Local Path", clique em "Choose..." para seleccionar a pasta de destino.
  5. Confirme: Clique em "Clone" para baixar o repositório.

Observação: Utilize sempre o método recomendado, Método 1: Pelo Navegador (Recomendado)

Diferença entre Clone e Fork:

  • Clone: Cria uma cópia local (no seu PC) de um repositório ao qual você tem acesso (ou pode contribuir directamente), mantendo uma ligação com o original para enviar (push) suas alterações.
  • Fork (Bifurcação): Cria uma cópia do repositório na sua própria conta do GitHub, ideal para projectos de outras pessoas, permitindo que você altere livremente e depois sugira suas mudanças via Pull Request (PR) para o repositório original.
