GitHub em vários PCs

Usar o GitHub em vários PCs envolve clonar seu repositório em cada máquina, fazer login na sua conta no GitHub Desktop ou CLI, e usar os comandos git add, commit, push (do PC 1 para o GitHub) e git pull/fetch (do GitHub para o PC 2) para sincronizar as alterações, sendo essencial configurar chaves SSH para segurança, especialmente em repositórios privados.

1. Preparação Inicial (No GitHub.com)

Crie seu repositório: Se ainda não o fez, crie seu repositório no GitHub.com.

2. No Primeiro PC

  • Instale o Git e GitHub Desktop: Baixe e instale o Git e o GitHub Desktop (opcional, mas recomendado).
  • Faça Login: Abra o GitHub Desktop e faça login na sua conta (File > Options > Accounts).
  • Clonagem: Clone o repositório do GitHub para o seu PC (File > Clone Repository) ou arraste a pasta do projecto.
  • Configure o Git: Defina seu nome e email (File > Options > Git).

3. No Segundo (e Próximos) PC

  • Repita os passos: Instale Git/GitHub Desktop e faça login na mesma conta
  • Clone novamente: Clone o mesmo repositório do GitHub para o novo PC.

4. Sincronização (O Fluxo de Trabalho)

  • No PC 1 (onde você codifica):
    • git add . (Adiciona as mudanças)
    • git commit -m "Sua mensagem" (Salva as mudanças localmente)
    • git push origin main (Envia para o GitHub).
  • No PC 2 (ou qualquer outro):
    • git fetch (Puxa as informações do servidor, mas não as mescla).
    • git pull origin main (Baixa e mescla as mudanças do GitHub para o seu PC).

5. Dica: Chaves SSH (para Repositórios Privados)

  • Se for usar repositórios privados, crie uma chave SSH em cada PC e adicione-a ao seu perfil do GitHub para autenticação segura, sem precisar digitar a senha toda hora.
