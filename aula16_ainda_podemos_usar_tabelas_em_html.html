Ainda podemos usar tabelas em HTML?

Sim, ainda podemos e devemos usar tabelas em HTML, mas exclusivamente para apresentar dados tabulares (linhas e colunas), como calendários, listas de preços ou cadastros, e não mais para organizar o layout da página, uma prática antiga e incorreta que foi substituída por CSS (Flexbox e Grid). Tabelas HTML são ótimas para clareza e acessibilidade quando o objetivo é exibir informações estruturadas.

Quando usar tabelas:

  • Dados Estruturados: Para exibir informações que naturalmente se encaixam em linhas e colunas, como:
    • Planilhas financeiras.
    • Horários e calendários.
    • Comparativos de produtos.
    • Listas de usuários ou informações de eventos.

    Quando NÃO usar tabelas:

    • Layout da Página: Para posicionar elementos visuais (como menus, barras laterais, etc.). Isso deve ser feito com CSS (Flexbox, Grid) para melhor acessibilidade e manutenção.

    Principais Tags para Tabelas:

    As tags principais para tabelas em HTML são <table> (a tabela em si), <tr> (linha da tabela), <td> (célula de dados) e <th> (célula de cabeçalho), que estruturam o conteúdo, com <thead>, <tbody>, <tfoot> e <caption> adicionando semântica e título, respectivamente, para organizar o cabeçalho, corpo, rodapé e título da tabela.

    Tags da Estrutura Básica

    • <table>: O contêiner principal da tabela.
    • <tr>: Define uma linha (Table Row).
    • <th>: Define uma célula de cabeçalho (Table Header).
    • <td>: Define uma célula de dados (Table Data).
      • Em resumo, use tabelas para dados, não para design, aproveitando o poder do HTML para estruturar e do CSS para estilizar.

    Tags de Semântica e Título (HTML5)

    • <thead>: Agrupa o conteúdo do cabeçalho da tabela.
    • <tbody>: Agrupa o conteúdo do corpo principal da tabela.
    • <tfoot>: Agrupa o conteúdo do rodapé da tabela (resumos, totais).
    • <caption>: Adiciona um título ou legenda à tabela, posicionada dentro das tags <table>.
