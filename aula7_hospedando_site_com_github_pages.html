Hospedando Site com GitHub Pages

Hospedar um site no GitHub Pages continua sendo uma das formas mais simples e gratuitas de colocar páginas estáticas (HTML, CSS e JS) online.

Passo a Passo Actualizado:

1. Preparar o Repositório

  • Nome do Repositório:
    • Para um site pessoal: Nomeie o repositório como seu-usuario.github.io. O site ficará disponível exactamente nesse endereço.
    • Para um projecto específico: Pode usar qualquer nome (ex: meu-projecto). O link será seu-usuario.github.io/meu-projecto/.
  • Visibilidade: O repositório pode ser público ou privado (se o seu plano permitir), mas o site resultante será sempre público na web.
  • Arquivo Principal: Certifique-se de que o arquivo principal do seu site se chama index.html e está na pasta raiz do projecto.

2. Activar a Hospedagem (GitHub Pages)

  1. No seu repositório no GitHub, clique na aba Settings (Configurações).
  2. No menu lateral esquerdo, na secção "Code and automation", clique em Pages.
  3. Em Build and deployment, sob a opção "Source", escolha Deploy from a branch.
  4. Em Branch, seleccione a branch principal (geralmente main ou master) e a pasta / (root).
  5. Clique em Save.

3. Acessar o Site

O GitHub levará alguns minutos para processar os arquivos. Após o processamento, um link aparecerá no topo da página de configurações do Pages indicando que o site está "Live".

Com o site publicado, poderá passar a URL aos seus amigos ou clientes para poderem ver o resultado dos seus estudos.

Se quiser criar QR Code e distribui-lo a quem o interessar, pode utilizar o site QR Code Generator. Cole a URL do seu site publicado e verá a imagem do QR Code.

Dicas Extras:

  • Domínios Personalizados: Você pode conectar um domínio próprio (ex: www.seusite.com) na mesma tela de configurações do Pages. Será necessário configurar os registros DNS (A ou CNAME) no seu provedor de domínio.
  • HTTPS: O GitHub Pages fornece certificados SSL automáticos e gratuitos. Active a opção Enforce HTTPS nas configurações para garantir que seu site seja seguro.
  • Geradores Estáticos: O serviço suporta nactivamente o Jekyll, mas você pode hospedar sites feitos com React, Vue, Next.js ou Hugo, desde que faça o "build" (geração dos arquivos estáticos) antes de enviar ou use o GitHub Actions para automatizar esse processo.
  • Limites: O site não pode exceder 1 GB de tamanho e possui um limite de largura de banda de 100 GB por mês.

Observação:

O GitHub Pages só permite hospedar HTML, CSS e Java Script.
