O que é Git e GitGub?

Git e GitHub são ferramentas complementares, mas com funções distintas no desenvolvimento de software:

Git (A Ferramenta)

O Git é um sistema de controle de versão distribuído. Ele funciona localmente no seu computador e serve para:

  • Registrar o histórico: Salva todas as alterações feitas nos arquivos por meio de "checkpoints" chamados commits.
  • Permitir reversões: Possibilita voltar a versões anteriores do projeto caso algo dê errado.
  • Trabalho em paralelo: Através de branches (ramificações), permite que você trabalhe em novas funções sem alterar o código principal até que esteja pronto.

GitHub (A Plataforma)

O GitHub é uma plataforma de hospedagem online baseada na nuvem que utiliza o Git como motor. Suas principais utilidades são:

  • Hospedagem de repositórios: Funciona como um "backup" na nuvem para o seu código Git.
  • Colaboração: Facilita o trabalho em equipe, permitindo que outros desenvolvedores visualizem, sugiram mudanças e contribuam com o projecto
  • Rede Social e Portfólio: É amplamente utilizado por recrutadores para avaliar as habilidades técnicas de um desenvolvedor através de seus projectos públicos.

Em suma, você usa o Git para gerenciar o seu trabalho local e o GitHub para compartilhar esse trabalho com o mundo ou com sua equipe.
