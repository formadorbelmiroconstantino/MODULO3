Gerenciando Repositório

Assim que tiveres um repositório criado para o projecto, todas as alterações que fizeres deverão ser guardadas localmente, primeiro, e depois no GitHub.

Note que depois da criação do seu repositório, no lado esquerdo, aparece um ícone novo parecido a um símbolo de rede com três bolinhas e toda a vez que fizer uma alteração num arquivo, vai aparecer uma letra M.

Guardar as Alterações de no Repositório, Localmente e no GitHub

Para actualizar seu projecto local e enviá-lo ao GitHub usando o GitHub Desktop, siga estes passos simples:

1. Visualizar as Alterações

Ao abrir o GitHub Desktop, ele detecta automaticamente os arquivos que você modificou ou adicionou na pasta do projecto.

  • No painel esquerdo, na aba Changes, você verá a lista de arquivos alterados.
  • Certifique-se de que os arquivos que deseja salvar estão marcados com o "check".

2. Guardar Localmente (Commit)

O "Commit" é o comando que grava as alterações no histórico do seu computador.

  • No canto inferior esquerdo, preencha o campo Summary (obrigatório) com uma breve descrição do que foi feito (ex: "Actualiza layout da home").
  • Clique no botão azul Commit to main (ou o nome da sua branch actual).

3. Enviar para o GitHub (Push)

Após o commit, as alterações estão salvas apenas na sua máquina. Para enviá-las ao servidor do GitHub:

  • Clique no botão Push origin que aparecerá na barra superior ou no painel central.
  • Se houver alterações feitas por outras pessoas no GitHub, o aplicativo pode pedir para você fazer um Fetch origin ou Pull primeiro para sincronizar antes de permitir o envio.

Como Excluir um Repositório Local?

Para excluir um repositório local, você tem duas opções principais, dependendo se deseja apagar apenas o vínculo com o Git ou excluir a pasta inteira do seu computador.

1. Remover apenas o "vínculo" com o Git (Manter os arquivos)

Se você quer que a pasta deixe de ser um repositório Git, mas quer manter os arquivos originais no seu computador, basta apagar a pasta oculta .git.

Nota: Se você não vir a pasta .git, é porque ela está oculta. No Windows, habilite "Itens Ocultos" no Explorador de Arquivos; no Mac, use Command + Shift + . (ponto).

2. Excluir a pasta completa (Apagar tudo)

Se o seu objectivo for remover o projecto e todos os seus arquivos permanentemente do disco rígido:

  • Pelo Explorador de Arquivos (Interface Visual): Clique com o botão direito na pasta do projeto e selecione Excluir (ou mova para a Lixeira).
  • No GitHub Desktop:
    1. Clique com o botão direito sobre o nome do repositório na lista lateral esquerda.
    2. Seleccione Remove...
    3. Na janela que abrir, marque a caixa que diz "Also move this repository to Trash" (Também mover este repositório para a Lixeira).
    4. Clique em Remove.

3. Remover apenas da lista do GitHub Desktop (Sem apagar arquivos)

Se você quer apenas "limpar" a lista de projetos do aplicativo GitHub Desktop, mas quer que a pasta continue existindo no seu PC:

  1. Clique com o botão direito no repositório dentro do aplicativo.
  2. Seleccione Remove...
  3. Não marque a caixa de seleção. Apenas clique no botão Remove. O repositório sairá do programa, mas os arquivos continuarão intactos na sua pasta.
Aviso: Diferente do GitHub.com, as exclusões locais feitas via comando rm -rf ou enviadas para a lixeira podem ser irreversíveis se a lixeira for esvaziada. Certifique-se de que não precisa de nenhum arquivo antes de exluir o projecto.

Como Excluir um Repositório no github.com?

Para excluir um repositório no GitHub, você deve acessar as configurações específicas do projecto. Esta acção é permanente e removerá todo o código, issues e comentários associados.

Siga estes passos:

  1. Acesse o GitHub e navegue até a página principal do repositório que deseja apagar.
  2. No menu superior, clique na aba Settings (Configurações).
  3. Na página que abrir (aba General), role até o final até encontrar a secção Danger Zone (Zona de Perigo).
  4. Clique no botão Delete this repository.
  5. O GitHub solicitará uma série de confirmações:
    1. Clique em I want to delete this repository.
    2. Clique em I have read and understand these effects após ler os avisos.
  6. Para confirmar que você está apagando o repositório correto, digite o nome completo do repositório (geralmente no formato seu-usuario/nome-do-repo) na caixa de texto indicada.
  7. Clique no botão final de confirmação para concluir a exclusão.

Observações importantes:

  • Permissões: Você precisa ser o proprietário ou ter permissões de administrador para realizar esta acção.
  • Repositório Local: Apagar o repositório no site do GitHub não apaga os arquivos que estão no seu computador. Para remover o vínculo local com o GitHub, você pode usar o comando git remote remove origin no seu terminal.

Alterar um Repositório de Privado para Público e Vice-Versa

Para alterar a visibilidade de um repositório de privado para público ou de público para privado no GitHub, siga estes passos:

  1. Acesse o GitHub e vá até a página principal do seu repositório.
  2. Clique na aba Settings (Configurações) no menu superior.
  3. Na aba lateral esquerda, certifique-se de que está em General (Geral) e role a página até o fim, na secção Danger Zone (Zona de Perigo).
  4. Ao lado de "Change repository visibility", clique no botão Change visibility.
  5. Seleccione a opção Make public.
  6. O sistema solicitará confirmações para garantir que você entende as consequências (como a perda de acesso a recursos exclusivos de repositórios privados, como certas ferramentas de CI/CD gratuitas):
    1. Clique em I want to make this repository public.
    2. Clique em I have read and understand these effects.
  7. Por fim, clique no botão de confirmação final (pode ser necessário digitar o nome do repositório ou sua senha/código 2FA para validar).

Atenção: Ao tornar um repositório público, qualquer pessoa na internet poderá visualizar e baixar o seu código. Verifique se não há senhas, chaves de API ou arquivos sensíveis (como .env) no histórico do repositório antes de realizar a alteração.

Dica de Segurança

Antes de começar a trabalhar no dia a dia, é recomendável clicar em Fetch origin para verificar se há atualizações no servidor e garantir que sua cópia local não esteja atrasada em relação à versão online.

Para mais detalhes visuais, você pode consultar o Guia oficial do GitHub Desktop.
