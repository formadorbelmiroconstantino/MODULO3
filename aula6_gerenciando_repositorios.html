Gerenciando Repositório

Assim que tiveres um repositório criado para o projecto, todas as alterações que fizeres deverão ser guardadas localmente, primeiro, e depois no GitHub.

Note que depois da criação seu repositório, no lado esquerdo, aparece um ícone novo parecido a um símbolo de rede com três bolinhas e toda a vez que fizer uma alteração num arquivo, vai aparecer uma letra M.

Guardar as Alterações de no Repositório, Localmente e no GitHub

Para actualizar seu projecto local e enviá-lo ao GitHub usando o GitHub Desktop, siga estes passos simples:

1. Visualizar as Alterações

Ao abrir o GitHub Desktop, ele detecta automaticamente os arquivos que você modificou ou adicionou na pasta do projecto.

  • No painel esquerdo, na aba Changes, você verá a lista de arquivos alterados.
  • Certifique-se de que os arquivos que deseja salvar estão marcados com o "check".

2. Guardar Localmente (Commit)

O "Commit" é o comando que grava as alterações no histórico do seu computador.

  • No canto inferior esquerdo, preencha o campo Summary (obrigatório) com uma breve descrição do que foi feito (ex: "Actualiza layout da home").
  • Clique no botão azul Commit to main (ou o nome da sua branch actual).

3. Enviar para o GitHub (Push)

Após o commit, as alterações estão salvas apenas na sua máquina. Para enviá-las ao servidor do GitHub:

  • Clique no botão Push origin que aparecerá na barra superior ou no painel central.
  • Se houver alterações feitas por outras pessoas no GitHub, o aplicativo pode pedir para você fazer um Fetch origin ou Pull primeiro para sincronizar antes de permitir o envio.

Dica de Segurança

Antes de começar a trabalhar no dia a dia, é recomendável clicar em Fetch origin para verificar se há atualizações no servidor e garantir que sua cópia local não esteja atrasada em relação à versão online.

Para mais detalhes visuais, você pode consultar o Guia oficial do GitHub Desktop.
